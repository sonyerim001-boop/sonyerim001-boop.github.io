## 📊 SQL 실습 문제 정리

### 🔹 기본 조회 및 조건 검색

- 문제 1: `SELECT *` → 전체 데이터 조회  
- 문제 2: 필요한 컬럼만 조회 → 효율적인 데이터 처리  
- 문제 3: `WHERE` → 특정 조건 필터링  
- 문제 4: 비교 연산자 (`>=`) 사용  
- 문제 5: `AND` 조건  
- 문제 6: `OR`, `IN` 활용  
- 문제 7: `LIKE` 문자열 검색  
- 문제 8: `BETWEEN` 범위 조회  

👉 다양한 조건문을 통해 원하는 데이터만 추출하는 방법을 학습하였다.

---

### 🔹 정렬

- 문제 9: `ORDER BY` 내림차순 정렬  
- 문제 10: 다중 정렬 (grade + score)

👉 데이터 정렬을 통해 결과를 구조적으로 확인할 수 있었다.

---

### 🔹 집계 함수

- 문제 11: `COUNT(*)` → 전체 개수  
- 문제 12: `SUM`, `AVG` → 합계 및 평균  
- 문제 13: `MAX`, `MIN` → 최대/최소값  

👉 데이터의 통계적 분석 방법을 이해할 수 있었다.

---

### 🔹 그룹화 및 조건

- 문제 14: `GROUP BY` → 그룹화  
- 문제 15: `HAVING` → 그룹 조건 필터링  
- 문제 16: `GROUP BY + ORDER BY`  

👉 데이터 집계 흐름을 실제로 확인하였다.

---

### 🔹 데이터 변경

- 문제 17: `UPDATE` → 데이터 수정  
- 문제 18: `DELETE` → 데이터 삭제  

👉 데이터 조작 명령어(DML)의 활용을 익혔다.

---

### 🔹 테이블 및 제약조건

- 문제 19: `CREATE TABLE` → 테이블 생성  
- 문제 20: `FOREIGN KEY` → 참조 무결성 유지  

👉 데이터베이스 구조 설계의 중요성을 이해하였다.

---

## 💡 핵심 정리

이번 실습을 통해 다음과 같은 흐름을 이해할 수 있었다.

- SELECT → WHERE → GROUP BY → HAVING → ORDER BY  
- 데이터 조회 → 정렬 → 집계 → 그룹화 → 조건 적용

👉 SQL은 단순 명령어가 아니라 **데이터 처리 흐름을 이해하는 것이 핵심**이다.

---

## ✍️ 느낀 점

SQL 문법을 암기하는 것보다 실제 데이터를 다루며 실행 흐름을 이해하는 것이 중요하다는 것을 느꼈다.

특히  
- WHERE vs HAVING  
- COUNT의 동작 방식  
- GROUP BY 처리 과정  

이 세 가지 개념이 실습을 통해 명확해졌다.
